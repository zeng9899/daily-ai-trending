验证安装：
```bash

```

## 🔐 第一步：获取 API Key

### 获取 RoxyBrowser API Key

1. 访问 [RoxyBrowser](https://roxybrowser.com/) 并注册账号
2. 登录后进入控制面板
3. 找到「API Key」或「API Token」选项
4. 点击「创建新密钥」
5. 复制生成的 API Key（格式类似：`2e4abf3ea18c5c9183f503f609487820`）

> ⚠️ 注意：API Key 只显示一次，请妥善保存！

### 获取智谱AI API Key

1. 访问 [智谱AI开放平台](https://open.bigmodel.cn/)
2. 注册账号并进行实名认证
3. 进入「API Keys」页面
4. 点击「创建 API Key」
5. 复制生成的 API Key

> 💡 新用户通常有免费额度，可以先试用

## ⚙️ 第三步：配置环境

### 3.1 安装全局依赖

```bash
# 安装 mcporter
npm install -g mcporter

# 验证安装
mcporter --version
```

### 3.2 配置 MCP 服务器

```bash
# 1. 复制配置模板
copy mcporter.json.example mcporter.json

# 2. 用记事本打开编辑
notepad mcporter.json
```

将文件内容修改为：
```json
{
  "mcpServers": {
    "roxybrowser-openapi": {
      "command": "npx",
      "args": ["-y", "@roxybrowser/openapi@beta"],
      "env": {
        "ROXY_API_KEY": "你刚才复制的RoxyBrowser API Key",
        "ROXY_API_HOST": "http://127.0.0.1:50000"
      }
    },
    "roxybrowser-playwright-mcp": {
      "command": "npx",
      "args": ["-y", "@roxybrowser/playwright-mcp@latest"]
    }
  }
}
```

### 3.3 配置 OpenClaw

```bash
# 1. 安装 OpenClaw（如果没有安装）
npm install -g openclaw

# 2. 复制配置模板
copy openclaw.json.example openclaw.json

# 3. 编辑配置文件
notepad openclaw.json
```

将文件内容修改为：
```json
{
  "models": {
    "providers": {
      "zhipu": {
        "baseUrl": "https://open.bigmodel.cn/api/paas/v4",
        "apiKey": "你刚才复制的智谱AI API Key",
        "api": "openai-completions",
        "models": [
          {
            "id": "glm-4.5-air",
            "name": "GLM-4.5-Air",
            "contextWindow": 200000,
            "maxTokens": 8192
          }
        ]
      }
    }
  },
  "agents": {
    "defaults": {
      "model": {
        "primary": "zhipu/glm-4.5-air"
      }
    }
  }
}
```

## 🚀 第四步：启动服务

### 4.1 启动 MCP 服务

```bash
# 在终端中运行
mcporter daemon start
```

应该看到类似输出：
```
✅ mcporter daemon started (PID: 12345)
```

### 4.2 启动 OpenClaw 网关

```bash
# 新开一个终端窗口
openclaw gateway start
```

应该看到类似输出：
```
🦞 OpenClaw Gateway started on http://127.0.0.1:18789
```

## 🔧 常见问题排查

### 问题1：mcporter 命令找不到

**解决方法：**
```bash
# 重新全局安装
npm uninstall -g mcporter
npm install -g mcporter

# 添加到环境变量（Windows）
set PATH=%PATH%;%APPDATA%\npm
```

### 问题2：API Key 错误

**症状：** 返回 "Invalid API Key" 或类似错误

**解决方法：**
1. 确认复制 API Key 时没有多余空格
2. 确认 API Key 没有过期
3. 登录对应平台重新生成

### 问题3：无法连接 GitHub

**症状：** 请求超时或连接失败

**解决方法：**
```bash
# 重启 MCP 服务
mcporter daemon stop
mcporter daemon start

# 检查网络代理设置
mcporter config list
```

### 问题4：端口被占用

**症状：** "Port already in use" 错误

**解决方法：**
```bash
# 查找占用端口的进程
netstat -ano | findstr :18789

# 结束进程（替换为实际PID）
taskkill /PID 12345 /F
```

## 🛡️ 安全建议

1. **不要提交真实 API Key 到 GitHub**
   - 使用 `.example` 模板文件
   - 将真实配置添加到 `.gitignore`

2. **定期更换 API Key**
   - 每1-3个月更换一次
   - 及时撤销泄露的 Key

3. **限制 API 配额**
   - 在平台设置中启用配额提醒
   - 防止意外超额
